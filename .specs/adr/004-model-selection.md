# ADR-004: Benchmark 先于模型选型

- **状态**: accepted
- **日期**: 2026-07-01
- **关联决策**: D10

## Context

NL-Shell 需要一个本地运行的轻量模型做意图分类和实体提取。候选模型包括 Qwen2.5-0.5B/1.5B、Gemma-2-2B、Phi-3-mini 等。不同模型在 ARM64 vs x86_64、中文理解能力、推理速度、内存占用方面差异很大。"拍脑袋选模型"风险高——选错了要推倒重来。

## Decision

**先建 benchmark，用数据选模型**。

Benchmark 要求（详见 AC-D6）：
- 1000 条中文 NL→结构化输出对
- 覆盖全部 12 个 intent，每个 ≥ 30 条
- 难度分布：easy 40% / medium 40% / hard 20%
- 含 100+ 对抗性样本

评测维度：

| 维度 | 指标 | 阈值 |
|---|---|---|
| 意图分类 | Accuracy | ≥ 85% |
| 实体提取 | F1 score | ≥ 80% |
| 推理延迟 | P99 latency (ARM64) | < 500ms |
| 内存占用 | RSS (含模型) | < 2GB |
| 跨架构 | x86_64 vs ARM64 intent 一致率 | 100% |

**选型流程**：建 benchmark → 候选模型跑分 → 输出对比报告 → 数据驱动选型。

**MVP 使用开源预训练模型 + prompt 工程**，不做微调。

## Consequences

- ✅ 避免"拍脑袋选模型"的常见陷阱
- ✅ benchmark 是持续资产（每次模型更新/替换都要跑）
- ✅ 数据驱动决策，有说服力
- ❌ 前期需要投入人力建设 benchmark（1000 条标注）
- ❌ 如果在 benchmark 建设期间模型生态发生重大变化（如新架构），可能需要重新评估
